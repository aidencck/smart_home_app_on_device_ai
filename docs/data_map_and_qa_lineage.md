# 智能家居端云协同 AI 系统数据地图与验证指导体系 (Data Map & QA Lineage)

> **文档状态**: 正式发布 (V1.0)
> **编写视角**: 数据专家 / 产品专家
> **目标读者**: 数据工程师、后端研发、端侧研发、测试团队 (QA)、产品经理
> **文档目的**: 作为项目的数据宪法，清晰界定全链路数据资产的边界、血缘关系 (Data Lineage) 以及脱敏/验证标准，指导研发和 QA 团队进行数据处理与测试验收。

## 1. 核心数据资产目录 (Data Asset Inventory)

在“端侧优先、云端兜底”的架构下，数据被严格划分为“端侧私有域”、“云端业务域”和“微调飞轮域”。

### 1.1 端侧私有域 (Edge Private Domain) - *最高机密*
数据仅保存在手机/中控屏的本地 Isar 数据库中，**默认绝不上云**。

| 数据实体 | 数据内容示例 | 生命周期 (TTL) | 脱敏与加密要求 | 业务用途 |
| :--- | :--- | :--- | :--- | :--- |
| **设备拓扑树** | 房间结构、设备MAC、设备物理位置 | 随设备绑定周期 | OS级加密 (AES-256) | 本地 GBNF 语法树生成 |
| **行为日志 (Behavior Log)** | `2023-10-01 19:00:00 打开主卧空调` | **30 天滚动删除** | OS级加密 (AES-256) | 本地 RAG 记忆与意图推断 |
| **原始语音/指令** | 用户的原生录音文件或输入文本 | **内存级，执行后即焚** | 无 (不落盘) | ASR 解析与大模型推理 |

### 1.2 云端业务域 (Cloud Business Domain) - *高可用与强一致*
数据保存在 FastAPI 后端的 PostgreSQL / Redis 中，用于兜底推理和状态同步。

| 数据实体 | 数据内容示例 | 存储位置 | 脱敏要求 | 验证边界 (QA) |
| :--- | :--- | :--- | :--- | :--- |
| **设备影子 (Device Shadow)** | `device_id: light_1, state: off, ts: 1710000000` | Redis Cluster | 仅包含状态，不含用户关联 | 过期时间 **0秒(高危)** 或 **60秒(普通)** |
| **AI 路由日志 (Routing Log)** | 请求耗时、路由决策(走本地/云端)、模型Token消耗 | PostgreSQL | **绝对禁止包含用户原始 Query** | 用于计费核算与模型命中率分析 |
| **账户与授权状态** | 用户ID、设备绑定关系、体验改善计划授权状态(Opt-in) | PostgreSQL | 手机号哈希存储 | 验证未授权用户数据绝对不进飞轮 |

### 1.3 微调飞轮域 (Data Flywheel Domain) - *合规红线*
经过严格筛选的失败交互日志 (Bad Cases)，用于 `model_forge` 微调。

| 数据实体 | 数据内容示例 | 存储位置 | 匿名化/脱敏要求 | 验证边界 (QA) |
| :--- | :--- | :--- | :--- | :--- |
| **脱敏遥测日志 (Telemetry)** | `{"query": "帮我把[PERSON]房间灯关了", "error": "intent_mismatch"}` | RabbitMQ -> Celery | 强制剥离 MAC/Phone，使用临时 Session ID | 端侧必须执行 NER 脱敏，剥离 PII |
| **微调数据集 (Hard Examples)** | JSONL 格式的指令-动作对 | OSS / MLX 训练机 | 经过大模型 (Judge) 二次过滤噪音 | 样本有效率需达到 10%-20% |

## 2. 全链路数据血缘与流转地图 (Data Lineage Map)

理解数据是如何在系统中流动的，是进行质量保证 (QA) 的基础。

```mermaid
flowchart TD
    subgraph 端侧 (Edge Domain)
        A[用户语音/文本] --> B(本地 NER 前置脱敏过滤)
        B --> C{本地/云端 路由决策}
        
        C --本地推理--> D[Isolate 本地大模型]
        D --> E[Isar 本地行为数据库]
        
        C --失败/云端兜底--> F[提取精简 Context]
    end

    subgraph 云端 (Cloud Domain - FastAPI)
        F --携带 Command ID--> G(API Gateway / Nginx)
        G --> H[FastAPI /api/v1/ai/chat]
        H --> I{Vector Clock 校验}
        I --过期--> J[触发 MQTT 探针拉取最新状态]
        I --有效--> K[vLLM / OpenAI 推理]
        K --> L((Redis: 设备影子))
    end

    subgraph 飞轮域 (Flywheel Domain)
        M[端侧拦截的 Bad Cases] --用户已授权--> N[FastAPI /api/v1/data]
        N --> O[(RabbitMQ)]
        O --> P[Celery Worker: LLM-as-a-Judge 清洗]
        P --剔除噪音--> Q[高质量 JSONL 微调数据集]
        Q --> R[model_forge LoRA 训练]
        R --OTA下发--> D
    end
```

## 3. 数据收集、处理与验证验收指导体系 (QA Guidelines)

为确保数据在流转过程中的**完整性、一致性和隐私合规性**，测试团队 (QA) 和研发团队需严格遵循以下指导体系进行验收。

### 3.1 端侧数据采集与本地处理验证 (Edge Data QA)
*   **测试目标**：确保本地数据不泄露，且生命周期受控。
*   **验收动作 (Checklist)**：
    *   [ ] **脱网黑盒测试**：断开手机网络，执行 50 条本地控制指令，抓包验证绝对没有任何行为日志或语音文件尝试向外发送。
    *   [ ] **存储加密审计**：Root / 越狱测试机，尝试导出 `smarthome.isar` 文件，验证其在未获取 OS 密钥时处于不可读的乱码状态。
    *   [ ] **TTL 滚动删除验证**：通过修改系统时间或注入模拟数据，验证超过 30 天的本地行为日志被系统后台任务彻底物理删除。
    *   [ ] **NER 前置脱敏验证**：构造包含“张三”、“李四”、“北京市朝阳区XX路”的指令触发云端兜底，抓包验证发往云端的 Payload 中，敏感信息已被 `[PERSON]` 或 `[LOCATION]` 替换。

### 3.2 云端设备影子与一致性验证 (Cloud State QA)
*   **测试目标**：杜绝“幽灵指令”和因状态不一致导致的物理安全隐患。
*   **验收动作 (Checklist)**：
    *   [ ] **高危设备 0 缓存测试**：针对门锁/烤箱类目，手动在 Redis 中篡改状态为“关闭”。发起开启指令，验证系统**强制下发 MQTT 探针**拉取真实状态，而不信任 Redis 数据。
    *   [ ] **并发状态覆盖测试 (Race Condition)**：使用 JMeter 并发模拟 1000 个设备同时上下线，验证带有 `last_update_ts` 的后发先至报文会被正确丢弃，不会覆盖最新的设备影子。
    *   [ ] **Command ID 防竞态测试**：模拟极端弱网（如 3G 网络），发起指令 A 后立即发起指令 B。验证云端指令 A 返回时，端侧通过比对 `Command ID` 成功将指令 A 静默丢弃，避免 UI 和设备动作错乱。

### 3.3 数据飞轮与微调合规验证 (Flywheel & Compliance QA)
*   **测试目标**：确保微调数据的高质量，并死守隐私合规红线。
*   **验收动作 (Checklist)**：
    *   [ ] **Opt-in 授权阻断测试**：在未勾选“体验改善计划”的设备上，人为制造交互失败 (Bad Case)。验证终端**完全不触发**向 `/api/v1/data/telemetry` 的网络请求。
    *   [ ] **标识符剥离验证**：审查到达 RabbitMQ 队列中的 JSON 消息体，验证绝对不包含 `MAC_Address`、`Phone_Number` 等物理追踪字段，仅包含临时 `Session_ID`。
    *   [ ] **LLM Judge 噪音清洗测试**：向数据清洗流注入 1000 条包含乱码、纯方言（无意义）、小孩子哭闹声转本的“脏数据”。验证 Celery Worker 的 LLM 过滤管道能成功丢弃 >95% 的脏数据，且不发生内存溢出 (OOM)。
    *   [ ] **一键被遗忘权测试 (Right to be Forgotten)**：在 App 端点击“注销并清除数据”，验证：1) 端侧 Isar 清空；2) 云端 PostgreSQL 账号注销；3) Redis 影子下线；4) 自动化脚本验证微调数据集 (JSONL) 中该用户的临时 Session 历史被物理删除（已合成进大模型权重的除外，需法务协议兜底）。

## 4. 异常处理与降级边界 (Boundary & Fallback)

数据地图必须明确当某一环节崩溃时，系统的容灾边界在哪里：

1.  **当 Redis 集群宕机**：
    *   *边界动作*：云端大模型兜底接口 (`/api/v1/ai/chat`) 拒绝执行任何控制类指令（返回：“状态获取失败，请重试”），但保留纯问答能力。
2.  **当 RabbitMQ/Celery 队列积压或宕机**：
    *   *边界动作*：`/api/v1/data/telemetry` 接口直接返回 `202 Accepted`，但内部执行丢弃逻辑（Drop data）。**绝不允许为了收集失败日志而阻塞核心的 API 网关**。
3.  **当云端大模型 API (OpenAI/vLLM) 欠费或超时**：
    *   *边界动作*：FastAPI 立即触发熔断，向端侧返回标准化错误码。端侧 Executor 拦截错误码并播报固定的本地兜底话术（如：“网络有点拥挤，请稍后再试”）。
