# 🧊 产品迭代愿景：从 "smartBQ" 到 "Ice"

> **“我们不是在制造一个遥控器。我们在制造一种‘随时都有完美冰块’的生活方式。”**

## 1. 哲学重构：推倒重来
我们将把原有的 `smartBQ`（智能制冰机伴侣）彻底推倒重来。那个名字太传统、太工业化了。从今天起，它的新名字叫 —— **“Ice (冰)”**。

我们的核心哲学是：**App 的存在，是为了让用户可以忘记它的存在。** 真正的智能不需要用户去“控制”，而是主动“预知”并“迎合”用户的生活节奏。

---

## 2. 核心迭代方向 (Product Pillars)

### 2.1 视觉与交互：极简主义的极致 (Zero-UI Concept)
*   **摒弃传统 UI**：干掉所有的汉堡菜单、设置项、Tab 栏。
*   **拟真物理映射**：打开应用，用户只会看到一个**极其逼真的、基于物理引擎的冰块与玻璃杯动画**。
    *   *状态映射*：冰槽满时，动画里的杯子就是满的；正在制冰时，屏幕上会呈现冰晶缓慢生长的微距动画。
*   **直觉式手势交互**：想制冰？不需要找“开始”按钮。唯一的动作：**用手指在屏幕上轻轻划入一颗冰块。** 就这样，它就开始工作了。

### 2.2 本地 AI 核心：预知能力 (Precognitive Intelligence)
*   **从“响应指令”到“预判需求”**：一个还需要你告诉它何时工作的“智能”设备，一点都不智能。
*   **习惯学习引擎**：利用我们的端侧 AI 与 Isar 数据库记录用户的行为节律。
    *   *场景示例*：系统学习到你“每周五晚上 7 点会喝一杯威士忌”。当周五 6:45 你走进家门（触发地理围栏或局域网连接）时，完美的冰块已经悄无声息地为你准备好了。
*   **零打扰后台运行**：无需每次打开 App，端侧 Agent 会在后台基于时间、温度、历史习惯自主决策并下发指令。

### 2.3 感官体验：听觉与触觉的闭环 (Sensory Design)
*   **拒绝工业电子音**：当冰块制好时，手机绝对不能发出那种廉价的、系统默认的“叮”声通知。
*   **高保真 ASMR 体验**：采用令人愉悦的、高保真录制的**“冰块落入玻璃杯的清脆撞击声”**。结合手机的高级线性马达（Haptic Touch），在冰块落下的瞬间给予干脆的震动反馈，完成产品体验的完美闭环。

## 3. 技术架构对齐 (Alignment with Current Architecture)

这个极其浪漫且极客的产品愿景，完美契合我们当前的**“端侧大模型 + Agent”**底层架构：

1.  **预知能力的基石 (Isar DB)**：我们现有的 `BehaviorLog` (行为日志) 不仅用于 RAG 问答，将升级为**习惯预测模型**的数据源，完全在本地计算，保护隐私。
2.  **复杂动画支持 (Flutter)**：Flutter 强大的跨平台渲染能力（Impeller 引擎）足以支撑我们在移动端实现 60fps 乃至 120fps 的流体与冰晶生长 Shader 动画。
3.  **后台静默控制 (Intent Executor)**：基于端云协同架构中的异步执行器，结合设备影子 (Device Shadow)，实现无需唤醒 App 也能精准控制硬件状态。

## 4. 下一步行动项 (Next Steps)
- [ ] **UI/UX 爆改**：移除现有的传统控制面板，引入 3D 渲染或高性能 2D 骨骼动画实现“冰块杯子”主视觉。
- [ ] **交互逻辑重写**：开发手势识别系统（Swipe to make ice）。
- [ ] **本地预判算法开发**：在端侧 Agent 中增加基于时间序列的习惯预测模块（Habit Prediction Module）。
- [ ] **音效与触觉工程**：采购或录制极致的 ASMR 冰块音效，并编写 Haptic 震动逻辑。
