# Smart Home App - 敏捷项目管理需求池 (Product Backlog)

> **文档状态**: 活跃 (Active)
> **视角**: 产品负责人 (Product Lead)
> **目标**: 将《从“被动控制”到“无感智能”》的产品愿景以及技术架构战略转化为可落地的史诗 (Epics) 和用户故事 (User Stories)，以便导入远程项目管理工具 (GitHub/Jira/GitLab)。

## 🌌 [EPIC-01] Zero-UI 交互与视觉重构 (Zero-UI Interaction & Visuals)
**业务价值 (Business Value)**：打破传统 IoT 产品的工业化控制面板设计，通过物理映射和直觉手势建立“自然如呼吸”的差异化体验壁垒。

*   **[STORY-1.1] 物理引擎驱动的设备状态映射**
    *   **作为** 智能家居用户
    *   **我希望** 告别繁琐的传统控制面板，通过极简的物理映射动画（如制冰机页面的冰块动画、咖啡机的流体动画）直观看到设备状态
    *   **以便于** 我获得更直观、不被打扰的视觉体验。
    *   *验收标准 (AC)*: 接入 Flutter Impeller，实现至少 3 款设备的 3D/Shader 物理映射 UI；移除传统的汉堡菜单和开关 Toggle。

*   **[STORY-1.2] 直觉式手势控制层**
    *   **作为** 智能家居用户
    *   **我希望** 通过直觉式手势（如将冰块划入杯中、轻触光晕）直接控制设备
    *   **以便于** 我不需要进入层层菜单寻找设置项。
    *   *验收标准 (AC)*: 开发通用手势映射中间件，将滑动/长按手势自动转化为底层 JSON 控制指令。

*   **[STORY-1.3] 多模态感官反馈 (ASMR & Haptics)**
    *   **作为** 智能家居用户
    *   **我希望** 在控制设备时能获得高保真的感官反馈（如清脆的冰块撞击声和干脆的线性马达震动）
    *   **以便于** 我的操作获得极其愉悦的非视觉确认。
    *   *验收标准 (AC)*: 废弃系统默认提示音，建立并集成平台级 ASMR 音效库与 CoreHaptics/Vibrator 触觉反馈参数。

## 🤖 [EPIC-02] 主动智能与场景模块进化 (Proactive Intelligence & Scenes)
**业务价值 (Business Value)**：实现从“响应指令”到“预判需求”的跨越，提升用户粘性和产品“智能感”。

*   **[STORY-2.1] 端侧习惯学习与主动推荐流**
    *   **作为** 智能家居用户
    *   **我希望** 系统能默默学习我日常操作设备的规律（时间、频率），并在场景页面顶部主动向我推荐卡片（如“晚安模式”）
    *   **以便于** 我可以一键采纳，减少手动配置的麻烦。
    *   *验收标准 (AC)*: 改造 `ScenesPage`，新增横向滑动的“AI 建议”区域；端侧模型能根据 Isar 日志准确输出推荐。

*   **[STORY-2.2] 弹性场景：动态感知执行**
    *   **作为** 智能家居用户
    *   **我希望** 触发场景时，系统能根据当前的环境因子（如当前室温、光照、季节）动态微调设备参数（如夏天开冷风，冬天开暖风）
    *   **以便于** 场景执行更符合当下真实需求，而不是死板的固定指令。
    *   *验收标准 (AC)*: 场景引擎支持变量占位符；执行时前置调用 AI Agent 获取当前环境上下文并微调参数。

*   **[STORY-2.3] 自然语言极简创建场景**
    *   **作为** 智能家居小白用户
    *   **我希望** 可以直接输入或语音说“帮我建一个看电影的场景，关灯然后打开电视”
    *   **以便于** 我能以最低门槛创建复杂的设备联动。
    *   *验收标准 (AC)*: 端侧模型具备实体解析和动作提取能力，自动生成配置表单供用户确认。

## 🔒 [EPIC-03] 隐私优先的端侧计算闭环 (Privacy-First On-Device AI)
**业务价值 (Business Value)**：打消用户对全天候感知的隐私顾虑，使“主动智能”具备落地的绝对前提。

*   **[STORY-3.1] 纯本地时序数据库画像**
    *   **作为** 极其看重隐私的用户
    *   **我希望** 我所有的设备操作日志和习惯分析都在手机本地处理，绝对不上传云端
    *   **以便于** 我能放心地让 AI 学习我的生活节律。
    *   *验收标准 (AC)*: 完善基于 Isar 的 `BehaviorLog` 架构，切断所有未经脱敏的日志上云网络请求。

*   **[STORY-3.2] 显性合规授权与数据飞轮开关**
    *   **作为** 智能家居用户
    *   **我希望** 清楚地知道哪些数据可能被用于改进模型，并能自主决定是否开启“体验改善计划”
    *   **以便于** 我对自己的数据拥有 100% 的控制权。
    *   *验收标准 (AC)*: App 首次启动强制弹出合规授权墙（Opt-in 且非默认勾选）；实现端侧轻量级 NER 剥离 PII（姓名/地址）。

## ⚡ [EPIC-04] 极速且无缝的端云协同体验 (Seamless Edge-Cloud Experience)
**业务价值 (Business Value)**：在本地算力与云端大模型之间建立完美平衡，兼顾响应速度与长尾能力。

*   **[STORY-4.1] 0 延迟本地推理控制**
    *   **作为** 智能家居用户
    *   **我希望** 日常的硬件控制能够做到极速响应（< 300ms），且生成指令格式 100% 正确
    *   **以便于** 获得丝滑的无感控制体验。
    *   *验收标准 (AC)*: 优化 Dart FFI 多线程调用 Llama.cpp 引擎；强制引入 GBNF 语法树杜绝 JSON 幻觉。

*   **[STORY-4.2] 长尾对话无缝云端降级**
    *   **作为** 智能家居用户
    *   **当我** 提出超出本地控制范围的复杂问题（如询问菜谱或深度闲聊）时
    *   **我希望** 系统能隐式切换到云端处理，不报错且无缝衔接
    *   **以便于** 享受全能的 AI 助手服务。
    *   *验收标准 (AC)*: 开发意图分类器 (Intent Splitting)；实现 FastAPI 云端兜底路由与 Semantic Cache 加速。

*   **[STORY-4.3] 多设备状态强一致性**
    *   **作为** 智能家居用户
    *   **我希望** 无论是在端侧预判执行还是云端异步返回，设备的状态都不会发生冲突或“幽灵跳动”
    *   **以便于** 获得稳定可靠的数字孪生体验。
    *   *验收标准 (AC)*: 在云端 Redis Cluster 引入 Vector Clock 机制；Flutter 端实现 Command ID 拦截器。
