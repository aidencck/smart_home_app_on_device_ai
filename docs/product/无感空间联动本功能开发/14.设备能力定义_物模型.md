# 设备能力定义（物模型前置准备）

**文档状态**: 草稿
**项目**: 卧室无感智能联动 (Zero-UI)
**背景**: 基于智能戒指实时生理数据的多设备无缝协同体验，梳理各设备需具备的能力，为后续物联网平台（IoT Platform）定义物模型（Thing Model / TSL: Thing Specification Language）提供业务输入。
**参考竞品**: Oura Ring Gen3, Ultrahuman Ring AIR, Eight Sleep Pod 4, Sleep Number 360, Philips Ambilight TV, Samsung Frame 等。

---

## 1. 智能戒指 (感知核心)

作为无感智能体系的数据输入源，主要负责生理指标的**高频监测**与**趋势分析**，强调超低功耗下的数据上报。

### 1.1 遥测数据与服务 (Telemetry / Properties / Services)
| 属性/服务名称 | 标识符 (Identifier) | 数据类型 (Data Type) | 业务说明及枚举值/单位 |
| :--- | :--- | :--- | :--- |
| **实时/静息心率** | `hr` / `restingHeartRate` | Integer | 单位：BPM (次/分钟)。包含实时心率及睡眠期间最低稳定心率。 |
| **心率变异性** | `hrv` | Integer | 单位：ms。反映自主神经系统状态的关键指标，如 45, 65, 50。 |
| **血氧饱和度** | `spo2` | Float | 单位：% (0.00-100.00) |
| **体表/皮肤温度** | `temp` / `skinTempDeviation`| Float | 单位：°C (如 36.6)。扩展：与个人基准体温的差值（如 +0.3）。 |
| **呼吸频率** | `respiration` | Float | 单位：brpm (次/分钟) |
| **睡眠阶段** | `sleepStage` | String (Enum) | 枚举：`0:清醒`, `1:浅睡(入睡期)`, `2:深睡`, `3:REM(快速动眼期)` |
| **睡眠效率** | `sleepEfficiency` | Integer | 单位：%。实际入睡时间/在床时间的比例。 |
| **准备度评分** | `readinessScore` | Integer | 范围：0-100。综合心率、睡眠、活动后的恢复建议。 |
| **昼夜节律相位** | `circadianPhase` | String (Enum) | 枚举：`0:波谷`, `1:上升期`, `2:波峰`, `3:下降期` |
| **体动状态** | `movement` | String (Enum) | 枚举：`轻微调整姿势`, `极少(深度沉浸)`, `逐渐活跃` 等 |
| **运动强度** | `activityIntensity` | String (Enum) | 枚举：`0:静止`, `1:低`, `2:中`, `3:高` |
| **实时压力水平** | `stressLevel` | Integer | 范围：1-100。基于HRV和皮肤电传导拟合。 |
| **经期预测** | `cycleTracking` | Struct | 包含：当前周期天数、预计排卵日预测。 |
| *(服务)* **强制同步**| `syncData` | Service | 调用后戒指开启高功耗蓝牙广播，推送本地缓存数据。 |
| *(服务)* **查找设备**| `findRing` | Service | 触发戒指内置微型LED闪烁或APP端距离感应。 |

## 2. 智能床 (执行终端)

作为物理环境的执行器，涵盖**多模态传感器（非接触式）**、**气动/机械执行器**以及**双分区环境控制**。

### 2.1 属性与服务 (Properties / Services)
| 属性/服务名称 | 标识符 (Identifier) | 数据类型 (Data Type) | 业务说明及枚举值/单位 |
| :--- | :--- | :--- | :--- |
| **背部抬升角度** | `headTiltAngle` | Integer | 单位：度 (°)。范围：0-60。典型值：`15`(阅读/零重力), `0`(平躺), `10`(唤醒)。 |
| **腿部抬升角度** | `footTiltAngle` | Integer | 单位：度 (°)。范围：0-45。用于促进血液回流。 |
| **零重力模式开关**| `zeroGMode` | Boolean | 开启后自动调整至背部15°、腿部30°等预设位。 |
| **分区支撑硬度** | `firmnessLevel` | Integer | 范围：1-100 (如SleepIQ)。对应左右侧气囊充气压力。 |
| **床体震动反馈** | `vibration` | String (Enum) | 枚举：`轻微背部舒缓`, `关闭`, `脉冲式线性马达` |
| **目标表面温度** | `targetSurfaceTemp` | Float | 单位：°C。支持左右侧独立分区控制（水暖或风暖）。 |
| **智能控温模式** | `autoTempPilot` | String (Enum) | 枚举：`0:关闭`, `1:睡眠诱导(升温)`, `2:深度睡眠(降温)`, `3:唤醒(升温)` |
| **暖脚功能强度** | `footWarmingLevel` | String (Enum) | 枚举：`0:关闭`, `1:低`, `2:中`, `3:高` |
| **在床状态** | `occupancyStatus` | String (Enum) | 枚举：`0:无人`, `1:左侧有人`, `2:右侧有人`, `3:全有人` |
| **鼾声检测水平** | `snoringLevel` | Integer | 单位：dB。检测到的实时噪音分贝。 |
| **打鼾干预开关** | `antiSnoreAutoTilt`| Boolean | 检测到打鼾超过阈值时，自动微抬头部角度（如10°-15°）。 |
| **翻身次数** | `tossTurnCount` | Integer | 单位：次。单次睡眠期间的体动频率。 |
| *(服务)* **智能唤醒**| `smartAlarm` | Service | 浅睡眠阶段通过轻微震动或缓慢抬升背部唤醒。 |
| *(服务)* **放松按摩**| `startMassage` | Service | 参数：部位(背/腿)、强度(1-3)、模式(脉冲/波浪)。 |

## 3. 智能电视 (视听环境)

在IoT场景下弱化传统“观影”属性，转变为**环境氛围核心**，侧重于**多屏联动**与**声光同步**。

### 3.1 属性与服务 (Properties / Services)
| 属性/服务名称 | 标识符 (Identifier) | 数据类型 (Data Type) | 业务说明及枚举值/单位 |
| :--- | :--- | :--- | :--- |
| **运行模式** | `operationMode` | String (Enum) | 枚举：`0:影视模式`, `1:艺术画廊(Frame)`, `2:环境信息流` |
| **屏幕画面** | `visual` | String (Enum) | 控制屏幕播放的Shader或内容。如：`篝火/雨滴`, `全黑息屏`, `日出光晕` |
| **画框感应开关** | `motionSensorArt` | Boolean | 艺术模式下，检测到无人时自动关闭屏幕以节能。 |
| **屏幕亮度** | `brightness` | Integer | 单位：% (0-100)。 |
| **环境光补偿** | `ambientLightComp` | Integer | 范围：0-100。根据环境光传感器自动调节屏幕背光。 |
| **音响系统** | `audio` | String (Enum) | 控制播放的内容。如：`ASMR 白噪音`, `静音`, `自然晨音` |
| **视听保护模式** | `eyeProtection` | Boolean | 开启滤蓝光及青少年模式。 |
| **氛围灯开关** | `ambilightSwitch` | Boolean | 控制背部/底座的环绕灯带。 |
| **灯光跟随模式** | `ambiSyncMode` | String (Enum) | 枚举：`0:跟随视频`, `1:跟随音频律动`, `2:静态色彩(如暖橘色呼吸)`, `3:晨曦渐变` |
| **灯光柔和度** | `ambiSoftness` | Integer | 范围：0-100。控制颜色切换的过渡平滑度。 |
| **全屋灯光联动** | `hueSyncState` | Boolean | 是否同步至全屋智能灯组 (如 Philips Hue)。 |
| **当前输入源** | `inputSource` | String (Enum) | 枚举：`0:TV`, `1:HDMI1`, `2:HDMI2`, `3:AirPlay`, `4:HomeKit` |
| *(服务)* **截图预览**| `screenCapture` | Service | 截取当前画面并推送到移动端App查看。 |
| *(服务)* **声场校准**| `roomAcousticCalib`| Service | 利用遥控器麦克风收集回声，自动优化空间音频声场。 |

## 4. 跨设备联动场景建议 (Rule Engine 侧写)

*为方便后续配置云端或边缘侧规则引擎，结合市场竞品提炼出更丰富的联动场景：*

*   **助眠联动 [戒指+床+电视] (PHASE 1)**：
    *   **触发**: 戒指检测到用户进入“准备入睡”状态，或手动开启。
    *   **动作**: 床体开启“暖脚模式”，目标表面温度调节至诱导睡眠的32℃，背部抬升15°(零重力)；电视画面=篝火/雨滴，开启ASMR，氛围灯=暖橘色呼吸(同步心率律动)。
*   **深睡氛围 [戒指+电视+床] (PHASE 2)**：
    *   **触发**: 戒指 `sleepStage` = "深睡期" (且心率、呼吸等指标平稳)。
    *   **动作**: 电视强制关闭所有指示灯(全黑息屏)、静音，切断氛围灯同步逻辑；床体平躺(0°)，震动关闭，开启脊椎强支撑，温度转为防冷热动态微调模式。
*   **起夜微光 [床+电视] (扩展场景)**：
    *   **触发**: 智能床 `occupancyStatus` 从“有人”变为“无人”（处于深夜时间段）。
    *   **动作**: 联动电视的氛围灯 (`ambilightSwitch`=True, `ambiSyncMode`=静态色彩) 开启20%亮度的暖黄色常亮模式，充当无感小夜灯；用户回床后自动关闭。
*   **防鼾闭环 [床+戒指] (扩展场景)**：
    *   **触发**: 智能床检测到 `snoringLevel` 超标，且戒指监测到血氧 `spo2` 下降波动。
    *   **动作**: 智能床触发 `antiSnoreAutoTilt`，微抬头部15°缓解气道受阻，并在清醒后于App生成“呼吸质量专项报告”。
*   **无感唤醒 [戒指+床+电视] (PHASE 3)**：
    *   **触发**: 设定闹钟时间临近（30分钟内），且戒指检测到 `sleepStage` = "浅睡眠" 或 `movement` = "逐渐活跃"。
    *   **动作**: 电视画面=日出渐变光晕，亮度随时间缓升至70%，音响=自然晨音，氛围灯=晨曦渐变；床体开启 `smartAlarm` 智能唤醒服务（脉冲式震动+背部缓抬至10°）。

## 5. 扩充的核心功能点总结 (基于市场竞品调研)

本章节记录了针对市面上顶级智能硬件（如 Oura Ring Gen3、Ultrahuman Ring AIR、Eight Sleep Pod 4、Sleep Number 360、Philips Ambilight TV、Samsung Frame 等）的深度调研结果。

### 5.1 智能戒指 (感知核心)
增加了更多生理维度的遥测指标和服务：
- `skinTempDeviation` (皮肤温度偏差)
- `sleepEfficiency` (睡眠效率)
- `readinessScore` (准备度评分)
- `circadianPhase` (昼夜节律相位)
- `stressLevel` (实时压力水平)
- `cycleTracking` (经期预测)
- `syncData` 和 `findRing` 控制服务

### 5.2 智能床 (执行终端)
从单纯的机械角度控制，扩充了**非接触式睡眠监测**和**更精细的温控与干预**：
- `headTiltAngle` / `footTiltAngle` (头/腿部独立抬升角度)
- `zeroGMode` (零重力模式一键开关)
- `firmnessLevel` (气囊支撑硬度调节)
- `targetSurfaceTemp` 和 `autoTempPilot` (双分区独立温控及智能睡眠诱导温控)
- `occupancyStatus` (在床状态监测)
- `snoringLevel` 和 `antiSnoreAutoTilt` (鼾声检测与自动打鼾干预微抬)
- `smartAlarm` (智能唤醒震动服务) 和 `startMassage` (放松按摩)

### 5.3 智能电视 (视听环境)
引入了更丰富的环境氛围与联动能力（借鉴 Frame 艺术电视与流光溢彩）：
- `operationMode` (影视/画廊/环境模式)
- `ambientLightComp` (环境光补偿背光)
- `ambilightSwitch`, `ambiSyncMode`, `ambiSoftness` (电视背部环境灯带控制与律动模式)
- `hueSyncState` (全屋灯光同步联动)
- `roomAcousticCalib` (空间音频声场自动校准服务)

### 5.4 扩展跨设备联动场景 (Rule Engine)
除原有的三阶段 (Phase) 外，补充了竞品中非常典型的复合场景：
- **起夜微光场景**：床体检测到无人 -> 电视开启 20%暖黄色环境光充当夜灯。
- **防鼾闭环场景**：床体检测到高分贝鼾声 + 戒指检测到血氧下降 -> 床体自动微抬 15° 缓解气道受阻。
