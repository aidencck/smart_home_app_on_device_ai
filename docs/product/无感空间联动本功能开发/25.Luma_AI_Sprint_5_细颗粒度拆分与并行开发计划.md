## 🚀 Sprint 5 并行开发总线 (Parallel Tracks)

基于 深度需求，Sprint 5 拆分为 **3 条主线**，涵盖前端交互重构、后端数据契约升级与底层硬件模拟器的数据格式对齐。

### 🧵 Track K: 底层传感器与数据瘦身 (分配至 `ai-integration-eng` 智能体)
**主线目标**: 改造硬件模拟器，使其遵循“原始数据不上云”及“基于 Semantic Data 触发”的规范。

| 子任务编号 | 细粒度任务描述 (Sub-Tasks) | 优先级 | 测试验收与跟踪标准 (AC) | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **K-1** | **重构 `ble_simulator.py` 数据 Payload**<br>更新智能床 (Bed) 与戒指 (Ring) 的广播格式，分离 `raw_metrics` 与 `computed_state`。 | P0 | 1. 戒指仅向外暴露 `sleep_stage`，不再广播心率绝对值。<br>2. 智能床增加 `vibration`, `support_level`, `angle` 等高阶属性。 | 待启动 |
| **K-2** | **模拟器端 Vector Clock 植入**<br>为每个模拟设备增加本地的递增版本号或时间戳 (`last_update_ts`)。 | P1 | 1. 广播包中带有 `vector_clock` 字段。<br>2. 每次状态变化（如检测到离床）时，时钟 +1。 | 待启动 |

### 🧵 Track L: 后端状态仲裁与防腐层升级 (分配至 `backend-architect` 智能体)
**主线目标**: 落实 Vector Clock 乐观锁控制，并在 API 层面对齐最新的统一物模型 JSON 契约。

| 子任务编号 | 细粒度任务描述 (Sub-Tasks) | 优先级 | 测试验收与跟踪标准 (AC) | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **L-1** | **统一物模型 JSON Schema 改造**<br>更新 `app/schemas/device.py`，使其严格适配 Lead 版文档中关于床 (`occupancy`, `angle`) 和灯光 (`shader_engine`, `transition_duration`) 的定义。 | P0 | 1. Pydantic 模型校验通过。<br>2. Swagger UI 显示最新的嵌套结构。 | 待启动 |
| **L-2** | **完善 `DeviceService.update_device_state` (乐观锁)**<br>加强基于 `vector_clock` 的 409 冲突返回逻辑，确保前端在冲突时能获取到云端的最新时钟。 | P0 | 1. 模拟旧时钟更新请求，被明确拒绝并返回最新时钟值。<br>2. 深睡硬锁拦截逻辑与新的 `angle` 字段路径完全兼容。 | 待启动 |
| **L-3** | **Override (人工干预) 日志与状态挂起**<br>提供 API 记录用户对 AI 的反向操作，并立即挂起 (Suspend) 该设备当前的自动化场景。 | P1 | 1. 用户手动操作床体时，生成一条 `Override_Log`。<br>2. 设备的影子 TTL 被重置，强行以物理状态为准。 | 待启动 |

### 🧵 Track M: 前端 React UI 商业级深度重构 (分配至 `frontend-architect` 智能体)
**主线目标**: 改造 `bedroom_zero_ui` 项目，使其脱离纯静态展示，引入商业级演示规范（Framer Motion 动效、乐观更新、玻璃拟态），真实对接后端的控制 API 和设备影子。

| 子任务编号 | 细粒度任务描述 (Sub-Tasks) | 优先级 | 测试验收与跟踪标准 (AC) | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **M-1** | **商业级环境视觉与骨架屏加载**<br>根据睡眠阶段动态切换背景色彩与动画（AWAKE/LIGHT_SLEEP/DEEP_SLEEP），引入 Skeleton 骨架屏消除白屏，并增加 Lucide 高清图标。 | P0 | 1. 背景渐变随 `sleep_stage` 平滑过渡。<br>2. 数据加载前显示标准骨架屏。 | 待启动 |
| **M-2** | **改造控制组件与乐观更新 (Optimistic Update)**<br>用户在界面拖动床体角度或切换场景时，UI 立即响应变化，后台静默携带 `vector_clock` 提交 PUT 请求。 | P0 | 1. 拖拽滑块无卡顿，请求异步发送。<br>2. 引入商业级 Toast 提示组件 (如 Sonner)。 | 待启动 |
| **M-3** | **前端深睡防惊醒交互 (Hard Lock) 与 409 回滚动效**<br>当状态为 `DEEP_SLEEP` 时，床体角度滑杆置灰加锁。若发生 409 冲突，滑块产生“回弹”或“震动”动效，并恢复为云端真实值。 | P0 | 1. 深睡期禁用滑块，点击出现“物理锁定”红色提示。<br>2. 409 冲突时触发明显的视觉回滚，Toast 警告“状态已被其他终端/AI更改”。 | 待启动 |
