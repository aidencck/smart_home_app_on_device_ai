# Luma AI - 商业级无感交互 UI/UX 深度设计与架构规范

**文档编号**: PRD-LUMA-UI-026
**文档状态**: Active
**核心目的**: 针对 `bedroom_zero_ui` 项目，从简单的 Demo 界面升级为具备“商业演示能力 (Commercial Demo Quality)”的工业级应用。基于“零交互 (Zero-UI)”、“状态随动”和“安全阻断”三大理念，重新定义视觉规范、动效反馈与异常处理机制。

---

## 1. 核心视觉理念 (Visual Concept)

商业级智能家居面板必须摒弃传统的“开关罗列”设计，转为“以用户状态为中心”的环境隐喻。

*   **Glassmorphism (玻璃拟态)**: 采用半透明毛玻璃材质，使背景随着“深睡/浅睡/清醒”的环境光（Ambient Light）发生柔和的色彩过渡，让用户直观感受到当前的空间状态。
*   **Zero-UI (隐去控制)**: 
    *   在用户清醒 (`AWAKE`) 时，展示丰富的环境控制（灯光色彩、床体支撑度）。
    *   在用户深睡 (`DEEP_SLEEP`) 时，界面自动“沉睡”，隐藏非必要滑块，仅展示极简的生理数据波形，防止光污染。
*   **Fluid Motion (流体动效)**: 所有的状态切换（包括数值变化、卡片展开/收起）必须具备 300ms-500ms 的平滑弹簧动效 (Spring Animation)，杜绝生硬的跳变。

---

## 2. 核心模块级设计 (Module Design)

### 2.1 动态环境视界 (Ambient Hero Section)
- **视觉层**: 占据顶部 1/3 区域的动态背景。
    - **AWAKE**: 清新明亮的渐变色 (如晨曦橙/天蓝)。
    - **LIGHT_SLEEP**: 柔和的暮光紫/深蓝。
    - **DEEP_SLEEP**: 极暗的深邃黑/星空蓝。
- **信息层**: 中央显示当前环境的“综合诊断”（如：“正在深睡护腰模式”，“空间已进入助眠状态”）。

### 2.2 生理数据光环 (Physiological Vital Ring)
- **设计元素**: 提取来自智能戒指的高阶语义数据（如心率/HRV/血氧等，若 API 可见）。
- **交互**: 采用 SVG 绘制的缓慢呼吸光环 (Breathing Halo)。当用户进入深睡时，光环的呼吸频率（Pulse Rate）随之变缓，形成视觉上的共鸣。

### 2.3 物理设备控制卡片 (Actuator Cards)
- **智能床 (Smart Bed)**:
    - **滑杆交互**: 采用带有刻度和回弹阻尼感的自定义 Slider。
    - **物理安全锁 (Hard Lock)**: 当处于 `DEEP_SLEEP` 状态时，滑杆上叠加“半透明锁”图标，整体变灰 (`opacity-50`)。用户尝试拖拽时，滑杆产生“震动拒绝”动效 (Shake Error)，并在底部弹出红色安全提示：“深睡期已物理锁定，防止惊醒”。
    - **高阶属性**: 增加“分区支撑度 (Firm/Soft)”和“震动反馈 (Vibration)”的胶囊切换按钮 (Pill Toggles)。
- **智能灯光 (Smart Light)**:
    - **日出唤醒 (Sunrise)**: 提供“渐变引擎 (Shader Engine)”的下拉/高亮选项，明确展示 15 分钟渐变的倒计时或进度感。

## 3. 商业级交互反馈机制 (Enterprise Feedback Loop)

一个商业级的系统必须能优雅地处理网络延迟、并发冲突和 AI 兜底。

### 3.1 乐观更新与回滚 (Optimistic Update & Rollback)
- 当用户拖动床头角度从 `0` 拖到 `30` 时，UI **立即**变为 30（消除网络带来的迟滞感）。
- 后台静默发送带 `vector_clock` 的 `PUT` 请求。
- **若成功**: 界面保持 30，后台静默更新本地 `vector_clock`。
- **若失败 (409 Conflict)**: 触发回滚动效，滑块从 30 自动“弹回”到服务器的真实值 (如 0)，同时顶部弹出红色 Toast 通知：“状态已被其他终端/AI 更改，已同步最新状态”。

### 3.2 骨架屏与优雅降级 (Skeleton & Fallback)
- **首屏加载**: 不允许出现白屏或生硬的 Loading 菊花图。必须使用与最终卡片排版一致的闪烁骨架屏 (Skeleton Screens)。
- **离线降级**: 若网关轮询失败超过 3 次，页面顶部出现黄色 Warning Banner：“边缘中枢连接断开，当前显示为缓存状态”，禁用所有控制操作，防止出现脏数据写入。

## 4. 技术选型建议 (Tech Stack for UI)
为了达成上述效果，前端 `bedroom_zero_ui` 需要引入以下技术栈：
- **Tailwind CSS**: 负责 Glassmorphism 和基础排版。
- **Framer Motion**: 负责所有组件的进出场动画、滑块的回弹以及 409 冲突时的“震动拒绝”动效。
- **Lucide React**: 提供统一、极简的商业级线框图标。
- **Sonner / React Hot Toast**: 负责商业级的 Toast 提示通知。
