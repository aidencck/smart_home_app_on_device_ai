## 🚀 Sprint 3 并行开发总线 (Parallel Tracks)

### 🧵 Track F: 边缘端生理特征解析与触发 (分配至 `ai-integration-eng` 智能体)
**主线目标**: 将戒指传输的原始体征（HR 心率、HRV、体动）解析为明确的睡眠事件（入睡、深睡、清醒），并推送事件到云端引擎。

| 子任务编号 | 细粒度任务描述 (Sub-Tasks) | 优先级 | 测试验收与跟踪标准 (AC) | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **F-1** | **智能戒指睡眠分期解析器 (`sleep_analyzer.py` / C++)**<br>通过滑动窗口读取模拟生理数据，判定 `AWAKE`, `LIGHT_SLEEP`, `DEEP_SLEEP` 状态。 | P0 | 1. 算法能够准确识别睡眠分期的转换（防抖，过滤噪声）。<br>2. 状态切换时向外发射标准化 Event。 | 待启动 |
| **F-2** | **边缘侧事件推送服务 (`event_dispatcher`)**<br>将上述状态转换包装为 JSON，通过 HTTP/MQTT 推送到后端 `POST /v1/automations/trigger`。 | P0 | 1. 推送格式包含 device_id, event_type, payload。<br>2. 具备弱网重试机制。 | 待启动 |

---

### 🧵 Track G: 云端自动化引擎与数据飞轮 (分配至 `backend-architect` 智能体)
**主线目标**: 提供自动化规则的 CRUD 管理，以及接收边缘端生理事件后，执行对应的场景或设备指令。

| 子任务编号 | 细粒度任务描述 (Sub-Tasks) | 优先级 | 测试验收与跟踪标准 (AC) | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **G-1** | **设计 Automation 自动化规则表结构**<br>定义触发条件 (Condition: e.g. event == 'DEEP_SLEEP') 和执行动作 (Action: e.g. turn off light)。 | P0 | 1. 数据库支持基于 JSON 的灵活条件表达式存储。<br>2. 支持启用/禁用状态。 | 待启动 |
| **G-2** | **开发事件触发与执行接口 (`POST /v1/automations/trigger`)**<br>接收生理事件，匹配规则，调用 `DeviceService` 发送控制指令。 | P0 | 1. 接口耗时 < 100ms。<br>2. 成功匹配并触发“深睡关灯”或“晨起日出”指令。 | 待启动 |
| **G-3** | **开发 AI Recommendation 模型与记录接口**<br>根据用户的 Override_Log 负样本，提供“建议推迟 10 分钟关灯”的 AI 推荐存储结构。 | P1 | 1. AI 推荐能被写入并推送到前端。<br>2. 包含 Accept / Ignore 两个状态闭环。 | 待启动 |

---

### 🧵 Track H: Flutter AI 推荐页面与确认逻辑 (分配至 `frontend-architect` 智能体)
**主线目标**: 为用户展示 AI 分析的睡眠报告及环境建议，并支持用户采纳（一键生成自动化）或忽略。

| 子任务编号 | 细粒度任务描述 (Sub-Tasks) | 优先级 | 测试验收与跟踪标准 (AC) | 状态 |
| :--- | :--- | :--- | :--- | :--- |
| **H-1** | **AI 页面 UI 骨架与推荐卡片设计**<br>实现报告概览区和建议卡片（如“我们发现您昨晚在浅睡期被灯光唤醒，建议将亮度调低 20%”）。 | P0 | 1. 视觉符合 PRD 要求的专业/深色科幻感。<br>2. 卡片提供“采纳”和“忽略”双按钮。 | 待启动 |
| **H-2** | **对接 AI Recommendation API 及采纳逻辑**<br>点击采纳后，调用后端接口自动创建或更新 Automation 规则。 | P0 | 1. 采纳成功后卡片显示为“已生效”。<br>2. Riverpod 状态自动刷新。 | 待启动 |
